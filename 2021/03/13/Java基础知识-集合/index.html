Java基础知识复习 -- 集合

集合

List顶层接口（可重复的集合）

实现类有：ArrayList、Vector、Stack、LinkedList

ArrayList

  • 实现了List接口的可扩容数组，它的内部是基于数组实现的。
  • 允许所有的元素，包括空值
  • 用在查询比较多，但是插入和删除比较少的情况
  • 线程不安全，如果多个线程至少有两个线程修改了ArrayList的结构，就会导致线程安全问题，作为替代条件可以使用线程安全的List

    LinkedList

  • 是一个双向链表，允许存储任何元素（包括 null ）
  • 线程不安全
  • 用在查询比较少，但是插入删除比较多的情况

Vector

  • 跟ArrayLis一样，都是基于数组实现的。
  • 它是线程安全的容器，它对内部的每个方法都简单粗暴的上锁
  • 同步方式需要的开销比较大

Stack

  • 继承于Vector类

Set顶层接口（不可重复的集合）

实现类有：HashSet、TreeSet、LinkedHashSet

HashSet

  • Set接口的实现类。
  • 线程不安全，如果需要并发访问HashSet，必须在外部加锁，或者使用Collections.synchronizedSet(new HashSet())进行加锁操作。
  • 支持fail-fast机制。

TreeSet

  • 是一个基于TreeMap的NavigableSet实现
  • 线程不安全，如果需要并发访问TreeSet，必须在外部加锁，或者使用Collections.synchronizedSet(new TreeSet())
  • 支持fail-fast机制。

LinkedHashSet

  • 继承于HashSet
  • 线程不安全，如果需要并发访问LinkedHashSet，必须在外部加锁，或者使用Collections.synchronizedSet(new LinkedHashSet())
  • 支持fail-fast机制。

Map顶层接口

实现类有：HashMap、TreeMap、Hashtable、LinkedHashMap、IdentityHashMap、WeakHashMap

HashMap

  • 是一个利用哈希表原理进行存储元素的集合
  • 允许的空的键，空的值
  • 线程不安全，如果需要线程安全操作，可使用Collections.synchronizedSet(new HashMap())
  • 支持fail-fast机制。

TreeMap

  • 是一个基于NavigableMap实现的红黑树，它会根据key自然排序，也可以通过Comparator进行定制排序。
  • 线程不安全，如果需要并发访问TreeMap，必须在外部加锁，或者使用Collections.synchronizedSet(new TreeMap())
  • 支持fail-fast机制。

Hashtable

  • 实现了一个哈希表，能够直接映射到值
  • 任何非空对象可以用作键和值。
  • 线程安全，如果需要多线程高并发推荐使用ConcurrentHashMap。
  • 支持fail-fast机制。

LinkedHashMap

  • 是Map接口的哈希表和链表实现的。
  • 线程不安全，如果需要并发访问LinkedHashMap，必须在外部加锁，或者使用Collections.synchronizedSet(new LinkedHashMap())
  • 支持fail-fast机制。

IdentityHashMap

  • 它跟HashMap的区别就是：IdentityHashMap采用“==”进行比较，而HashMap则采用“equals()”进行比较
  • 线程不安全
  • 支持fail-fast机制。

WeakHashMap

  • 基于哈希表的Map基础实现，带有弱键。
  • 支持空键、空值
  • 支持fail-fast机制。
  • 经常作缓存
  • 不允许重复
